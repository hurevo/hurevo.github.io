This document provides essential information for agentic coding assistants operating in this codebase.

```bash
bun run lint       # Run ESLint checks
bun run dev        # Start development server
bun run preview    # Preview production build
```

## Code Style Guidelines

### Imports & Organization
- **Import Order**: External dependencies → Local imports → Type imports
- **Path Aliases**: Use `@/*` for imports from `src/` directory
  ```typescript
  import { Button } from '@/components/ui/button'
  import { cn } from '@/lib/utils'
  ```
- **Type Imports**: Use explicit `type` keyword for type-only imports
  ```typescript
  import type { VariantProps } from 'class-variance-authority'
  ```

### TypeScript Configuration
- **Target**: ES2022 with strict mode enabled
- **Module**: ESNext with bundler resolution
- **Strict Checks**:
  - `strict: true` - All strict type-checking flags enabled
  - `noUnusedLocals` - Unused variables cause errors
  - `noUnusedParameters` - Unused parameters cause errors
  - `noUncheckedSideEffectImports` - Side effects must be declared
  - `noFallthroughCasesInSwitch` - Switch fallthrough detection
- **Path Resolution**: Always use `@/` for src imports

### Formatting & Naming
- **Semicolons**: Required (enforced by ESLint)
- **Components**: PascalCase (e.g., `HeroSection`, `Button`)
- **Functions**: camelCase (e.g., `getLocalizedUrl`, `createRouter`)
- **Constants**: UPPER_SNAKE_CASE for true constants
- **Files**: Kebab-case for utilities, PascalCase for components

### React & Components
- **React 19.2.0** in use with latest hooks
- **Functional Components**: Use only function components
- **Hooks**: Follow react-hooks linting rules (all hooks must be in dependency arrays)
- **Component Pattern**: Use variant systems with CVA (class-variance-authority)
  ```typescript
  const buttonVariants = cva("base-styles", {
    variants: { /* variants */ },
    defaultVariants: { /* defaults */ }
  })
  ```

### Key Libraries
- **UI Components**: Base UI React + shadcn/ui pattern
- **Styling**: Tailwind CSS v4 with `@tailwindcss/vite` plugin
- **Routing**: TanStack Router with auto-generated route trees
- **i18n**: Paraglide JS for localization
- **Icons**: Lucide React + Radix UI icons
- **Utilities**: `clsx`, `tailwind-merge` for className merging

### Error Handling
- Use TypeScript's strict null checking (`strict: true`)
- Always handle potential null/undefined with proper type guards
- Use `?.` optional chaining and `??` nullish coalescing
- Return meaningful error states rather than throwing in components
- Log errors appropriately (implementation-specific patterns in codebase)

### File Structure
```
src/
  components/
    ui/              # Base UI components (Button, Card, etc.)
    sections/        # Page sections
    seo/            # SEO-related components
  lib/               # Utilities (utils.ts)
  paraglide/        # Auto-generated i18n (DO NOT EDIT)
  main.tsx          # App entry point
  App.tsx           # Root component
  index.css         # Global styles
  routeTree.gen.tsx # Auto-generated routes (DO NOT EDIT)
```

### Auto-Generated Files
- **`routeTree.gen.tsx`** - Generated by TanStack Router, do not edit
- **`paraglide/`** - Generated by Paraglide i18n, do not edit
- These are recreated during development; changes will be lost

### Common Patterns

**Component with Variants**:
```typescript
import { cva, type VariantProps } from 'class-variance-authority'
import { cn } from '@/lib/utils'

const componentVariants = cva('base-classes', {
  variants: { variant: { default: 'default-styles' } },
  defaultVariants: { variant: 'default' }
})

function Component({ className, variant, ...props }: React.HTMLAttributes<HTMLElement> & VariantProps<typeof componentVariants>) {
  return <div className={cn(componentVariants({ variant, className }))} {...props} />
}
```

**Router Integration**:
- Use `@tanstack/react-router` with auto-generated route tree
- Route files automatically picked up from `src/routes/` structure
- Routes are type-safe through TypeScript integration

## ESLint Configuration
- **Extends**: ESLint recommended + TypeScript ESLint + React Hooks + React Refresh
- **Rules Enforced**: 
  - React hooks dependency arrays (via `react-hooks` plugin)
  - No unused variables or imports
  - Proper TypeScript typing
  - Vite/React Refresh compatible code

## Key Notes for Agents
1. **Always run `bun run build`** before considering work complete to catch type errors
2. **Use strict types** - embrace TypeScript's strictness, don't use `any`
3. **Follow component patterns** - use CVA for styled variants consistently
4. **Path aliases** - never use relative imports like `../../../`, always use `@/`
5. **React 19** features available - use latest patterns and hooks
6. **i18n aware** - understand that URLs are localized; check Paraglide integration
